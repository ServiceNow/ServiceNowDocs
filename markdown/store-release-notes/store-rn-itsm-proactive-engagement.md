# Proactive Engagement release notes

Version history for the Proactive Engagement application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 2.5.1 - March 2025**

    Minor bug fixes related to Access Control List \(ACL\) and application creation.

-   **Version 2.5.0 - February 2025**
    -   New:
        -   Proactive Engagement on Desktop Assistant: Provide self-help instructions or notifications using Desktop Assistant as the notification channel.
        -   New use cases for Proactive Engagement: Support employees to resolve their issues such as poor Wi-Fi connectivity, device crash, app crash, and disconnected Zscaler connectivity.
-   **Version 2.0.0 - November 2024**
    -   New improvements for SR - ITSM - Proactive Engagement - v2.0 are listed below:
        -   Added four new use cases required for your organization to meet your specific needs.
            -   Windows disk clean-up remedial action for low disk space.
            -   Battery replacement catalog request for poor device battery health.
            -   Self-help instructions to enable VPN or Zscaler.
            -   Notify or nudge when the device is not rebooted for a period of time.
        -   Introduced new notifications channels and user engagement settings to notify your end-users.
        -   Added support for Flow-based issue detection in Proactive Engagement, allowing detection and resolution of issues that cannot be detected using standard metric rules.
        -   Added experience issue to incident origin and interaction summary in work notes.
-   **Version 1.0.0 - August 2024**

    Proactive Engagement enhances employee digital experience, productivity, and satisfaction by allowing them to self-solve daily digital issues seamlessly without needing the organization to share all solution sets.
