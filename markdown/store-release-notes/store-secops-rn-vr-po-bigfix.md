# Vulnerability Response Patch Orchestration with HCL Bigfix release notes

Version history for the Vulnerability Response Patch Orchestration with HCL Bigfix integration on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 1.3.0 - May 2025**

    New: The patch orchestration data model is extended to integrate with ITSM systems seamlessly.

-   **Version 1.2.1 - November 2024**

    Fixed: The Patch Orchestration with HCL Bigfix Integration sends the correct source\_id during import for a patch deployment instead of the serial\_number.

-   **Version 1.1.4 - August 2024**

    Fixed: Improvements to address Big Fix Integration run failure: "Error: TypeError: Cannot read property "import\_starttime\_buffer" from undefined"".

-   **Version 1.1.3 - November 2023**

    Minor fixes for this release.

-   **Version 1.1.0 - August 2022**

    Changed: Updates to ACLs.

-   **Version 1.0.1 - March 2022**

    This patch orchestration integration with the Vulnerability Response application uses scheduled patch imports \(Fixlets\) from the HCL BigFix product to help your security and IT teams identify your critical vulnerabilities and then apply the updates that can fix them.
