# Jenkins v2 spoke release notes

Version history for the Integration Hub Jenkins v2 spoke on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 1.4.0 - June 2026**

    Fixed: Streamline ACLs.

-   **Version 1.3.0 - April 2026**

    Added 5 new external triggers

-   **Version 1.2.0 - February 2023**

    New: Authentication template for spoke setup

-   **Version 1.1.4 - September 2022**

    Patch release of Jenkins v2 Spoke for IntegrationHub. This version includes a fix to improve the installation performance of the spoke.

-   **Version 1.1.1 - December 2021**

    New: This version adds 5 new actions \(Look up Builds by Date Range, Look up Commits by Build, Look up State Nodes by Build, Look up Stages by Builds, Look up Test Report by Build\).

-   **Version 1.0.4 - August 2021**

    This is a patch release of Jenkins v2 Spoke for IntegrationHub. It includes KMF module for additional security of password2 fields.

-   **Version 1.0.3 - June 2021**
    -   This is a patch release of Jenkins v2 Spoke. It includes the below changes:
        -   Fix for missing complex object for a sub-flow.
        -   Updated error handling in "Perform build with parameters".
        -   Fix for sub-flow for "Perform build with parameters".
-   **Version 1.0.2 - February 2021**

    Jenkins V2 1.0.2 is a patch release in which the fix is provided to address the issue with missing output in the Perform Build action.

-   **Version 1.0.1 - April 2020**
    -   The Integration Hub Jenkins v2 spoke helps to get the necessary build information from the continuous integration pipeline which can be stored in ServiceNow to understand build statistics over time. The data retrieved from Jenkins can then be curated, and additional workflows can be kicked off based on the policies defined.
    -   The Jenkins spoke can also be used for job and build management. For example, you can create a Jenkins job and initiate a build from your ServiceNow instance.
