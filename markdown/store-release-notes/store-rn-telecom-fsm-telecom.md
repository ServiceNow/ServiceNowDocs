# Field Service Management for Telecommunications release notes

Version history for the Field Service Management for Telecommunications application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 4.1.0 - June 2026**
    -   Fixes related to Accessibility
    -   Fixes related to zoom in/out an reflow
-   **Version 4.0.0 - March 2026 \(Australia\)**

    Updated dependent version of TMF API to ensure compatibility.

-   **Version 3.0.2 - December 2025**
    -   Supported below security directives:
        -   Implement granular admin roles
        -   Dot-Walk Scoping Bypass
        -   Implement Java 21 Runtime Migration
        -   Read Only field Security
-   **Version 2.0.0 - August 2025**
    -   New:
        -   Support TMF 697 Inbound API for Work Order Management
        -   Support GET/LIST/POST/PATCH/DELETE operations for Work Order API.
-   **Version 1.5.0 - February 2025**

    No changes in the version 1.5.0.

-   **Version 1.1.0 - November 2024**

    The ServiceNow Field Service Management for Telecommunication \(FSMT\) application enables you to manage the field service management API for the Telecom customers. Use this application to streamline the appointment booking, service delivery, and assurance processes. The ServiceNow Field Service Management for Telecommunication \(FSMT\) application integrates with the Field Service Management \(FSM\) to manage and streamline appointment workflow for the telecommunications industry. The FSMT supports the TM Forum-aligned Open APIs and their functionalities that enable an open digital ecosystem. These APIs promote faster integration with Field Service Management to manage appointment booking, service delivery, and assurance processes.
