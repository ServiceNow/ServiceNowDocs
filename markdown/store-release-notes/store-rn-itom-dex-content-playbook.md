# DEX Content Playbook release notes

Version history for the DEX Content Playbook application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 3.1.0 - August 2025**

    See [DEX Application and Device Health](https://www.servicenow.com/docs/bundle/store-release-notes/page/release-notes/store/it-operations-management/store-rn-itom-dex-app-device-health.html) product for release notes. This app is a dependency of DEX Application and Device Health.

-   **Version 2.5.0 - February 2025**
    -   New:
        -   macOS metrics: System crashes, Application crashes, VPN details, Energy consumption, and Power consumption
        -   Windows metrics: Windows registry keys, SCCM, list of executables, and BIOS details
    -   Changed: macOS metrics: System memory details, BitLocker details, and Hard Drive status
-   **Version 2.4.0 - November 2024**
    -   New:
        -   Metrics:
            -   macOS metrics: Support System compliance
            -   Windows metrics: Memory details, Incoming network bytes, Outgoing network bytes, Domain network details, Domain network route details, Network adapter details, Network connection profiles, System compliance, CPU performance details, Device details processor ID, OS details, Memory details - virtual memory VPN status
        -   Actions:
            -   macOS actions: Restart service
            -   Windows actions: Disk clean up, Restart service, and Clear app cache
-   **Version 2.3.0 - August 2024**
    -   Changed:
        -   Metric rule integration with Proactive Engagement
        -   Device metric UI improvements
-   **Version 2.2.3 - June 2024**
    -   Fixed:
        -   Metric Rule Improvements
        -   Performance Improvements
-   **Version 2.2.1 - May 2024**
    -   The DEX Content Playbook provides the content to assist with monitoring applications and devices, as well as offer support for remediating identified experience issues.
    -   A content playbook helps you to interpret new data from SaaS and local-to-the-device installed application monitoring, as well as health monitoring from the employee's device itself.
    -   Monitoring encompasses the measurement of application performance and availability, as well as device system characteristics, to guarantee a positive digital experience for end-users.
