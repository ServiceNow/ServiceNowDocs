# Triggers release notes

Version history for the Triggers application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 28.1.4 - May 2026 \(Zurich\)**

    New: Workflow Studio as a Designer source for triggers generated within Workflow Studio.

-   **Version 29.0.2 - March 2026**
    -   New: Business calendar trigger is available.
    -   Fixed:
        -   Accessibility issues fixed.
        -   UI issues fixed.
-   **Version 28.1.3 - August 2025**
    -   New Bug fixes in this app version include the following:
        -   UI improvements, including theming updates for Triggers
        -   Accessibility improvements
        -   Localisation fixes
        -   UI fixes for advanced scheduled triggers
-   **Version 27.2.5 - May 2025**
    -   Fixed:
        -   Minor UI fixes
        -   Translations
        -   Accessibility
        -   Theming support
-   **Version 27.1.5 - February 2025**
    -   Save a set of trigger definitions as a reusable trigger. Enable flow authors to select the saved trigger from some or all application flows. Specify whether flow authors can see the trigger details or add conditions to the trigger.
        -   Save a set of trigger definitions as a reusable trigger that responds to an external event through webhooks and record based events.
        -   When an event happens in the configured third-party application that meets the specified conditions, the trigger is activated.
