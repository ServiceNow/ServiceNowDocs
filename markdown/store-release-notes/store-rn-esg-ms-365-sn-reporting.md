# Microsoft 365 for ServiceNow Reporting release notes

Version history for the Microsoft 365 for ServiceNow Reporting application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 22.3.1 - June 2026 \(Australia\)**
    -   New:
        -   M365 reporting as a development plugin — Microsoft 365 reporting has been restructured as a standalone development plugin, improving modularity and deployment flexibility.
        -   AI-generated content links management — The Manage Links page now displays AI-generated content links, providing a centralized view of all AI-created references within reports.
    -   Changed:
        -   Business domain reference update — Updated the data model to reference the new GRC business domain structure as part of the Document Designer consolidation.
        -   Accessibility improvements — Icons and buttons in the O365 Add-in have been updated to meet minimum target size criteria for WCAG 2.2 compliance.
    -   Fixed: Business domain editing for ESG admin — Fixed an issue where ESG administrators were unable to edit Business Domain and Roles \(read-only\). Also resolved an issue where deleting a business domain left linked role mapping records with empty domain values.
-   **Version 22.0.1 - March 2026**
    -   Fixed additional filters usage for reference fields.
    -   Optimized performance for showing additional filters in the add-in.
-   **Version 21.1.1 - December 2025 \(Zurich\)**

    New: Added functionality to link claims created via the Microsoft 365 add-in to their corresponding disclosure reports.

-   **Version 21.0.1 - August 2025**

    Enabled creating claims directly from Microsoft Word to ServiceNow via the Microsoft 365 plugin.

-   **Version 20.1.1 - May 2025**

    Fixed: Important security fixes.

-   **Version 20.0.1 - February 2025**
    -   New:
        -   Dynamic filtering for Microsoft 365 for ServiceNow Reporting.
            -   Filter the fields dynamically and set up dependencies by using related fields. In the Microsoft 365 add-in, you can configure fields so that cascading filters are supported dynamically. You can select a value in a field and have the related fields automatically updated to show the relevant options. This process helps you to streamline data entry and improve efficiency.
    -   Fixed: Minor improvements and bug fixes related to HTML content within the document.
-   **Version 19.1.0 - November 2024**
    -   Fixed:
        -   Improved workspace experience for users with low vision or those who rely on assistive technology, such as screen readers. This includes optimizing zoom capabilities.
        -   Security fixes.
-   **Version 19.0.2 - August 2024**
    -   New:
        -   Added support for explicit logout functionality
        -   Added support for PA visualizations
        -   Added the ability to copy disclosure number from add-in
-   **Version 18.0.1 - February 2024**

    Microsoft 365 for ServiceNow Reporting provides reporting capabilities to organisational reporting managers to seamlessly report on ServiceNow data and reports through Microsoft Word and establish an auditable link between the document and the data.
