# Retail In-Store Operations release notes

Version history for the Retail In-Store Operations application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 4.8.0 - June 2026**
    -   New:
        -   Case type standardizes the reporting, tracking, and resolution of in-store issues with structured task assignments, reducing time to resolution.
        -   View Store Case page with tabbed layout and task view.
        -   Store case state management, details, and field updates.
        -   HQ task state management from the store view.
-   **Version 4.7.0 - June 2026 \(Zurich\)**
    -   Fixed:
        -   Case type standardizes the reporting, tracking, and resolution of in-store issues with structured task assignments, reducing time to resolution.
        -   Updated the View Store Case page with a tabbed layout and task view.
        -   Improved store case state management, details, and field updates.
        -   Enabled HQ task state management from the store view.
-   **Version 4.2.0 - May 2026**

    New: Added capability to track the status of individual store tasks within each store case on the workspace

-   **Version 3.4.0 - May 2026 \(Zurich\)**

    New: Added capability to track the status of individual store tasks within each store case on the workspace

-   **Version 3.1.0 - March 2026 \(Australia\)**

    New: Store associates and managers will have the ability to fulfil the store cases and tasks generated from store plans.

-   **Version 3.0.0 - March 2026 \(Zurich\)**

    New: Store associates and managers will have the ability to fulfil the store cases and tasks generated from store plans.

-   **Version 2.1.0 - August 2025**

    The Retail In-store Operations application allows store team members to report and track in-store operational issues, whether for routine or cyclical demands. This ensures that issues are documented and monitored for consistent execution and support.
