# Shift Handover Application release notes

Version history for the Shift Handover Application application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 1.8.0 - June 2026**
    -   Fixed:
        -   Fixed the improper handling of DST in Shift handover log creation.
        -   Fixed an issue where 'Change to "In Progress" State' button wasn't working.
-   **Version 1.7.1 - March 2026**

    Fixed: Fixed the issue where the skip request was not supported by the AI Agent and prompted the user to perform actions which the user did not expect.

-   **Version 1.7.0 - December 2025**
    -   Fixed: The Skip request was not supported by the latest AI Agent. Instead, it prompted the user to perform the action, which didn't align with user expectations, along with other observed issues.
    -   Changed: Read only field changes.
-   **Version 1.0.61 - September 2025**

    Fixed: Record mention links were not opening in the shift handover form page.

-   **Version 1.0.5 - August 2025**

    New: Rte component incorporating start date and end date, count feature, dynamic record mention.

-   **Version 1.0.3 - May 2025**

    Fixed: Localization bug has been resolved as part of this release.

-   **Version 1.0.2 - July 2024**

    Changed: The Shift Handover\(sn\_escm\_sh\)is designated as a dependent application for "Security Incident Response Workspace \(sn\_si\_aw\)". Therefore, you can't install it directly.

-   **Version 1.0.1 - May 2024**

    The Shift Handover application will help analysts working in shifts hand over important work items to the next shift.
