# Ansible spoke release notes

Version history for the Integration Hub Ansible spoke on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 2.3.0 - June 2026**

    Fixed: Streamline ACLs.

-   **Version 2.2.9 - December 2024**

    Fixed: Authentication template.

-   **Version 2.2.8 - September 2023**

    Fixed: The license requirement.

-   **Version 2.2.7 - September 2022**

    Fixed: Patch version of Ansible Spoke for Integration Hub. This version includes fixes to improve the installation performance and to avoid a change in the action, LookUpUsersById

-   **Version 2.2.5 - February 2022**

    Fixed: Patch release of Ansible Spoke for IntegrationHub. This version includes a fix for the extra variable not getting populated in Ansible UI.

-   **Version 2.2.4 - November 2021**

    Patch release of Ansible Spoke for IntegrationHub. This version includes fixes for localization warnings.

-   **Version 2.2.2 - October 2021**

    Fixed: Security bug.

-   **Version 2.2.1 - September 2021**
    -   New:
        -   Added inbound webhook support framework
        -   Added sample remote table
        -   Added sample decision table framework
        -   Added authentication template
    -   Changed: Categorized the actions more meaningfully for easier search
-   **Version 2.0.1 - August 2021**

    This is a Patch Release of Ansible spoke for IntegrationHub. This version adds KMF module for additional security at password2 fields.

-   **Version 1.0.2 - February 2021**

    Ansible 1.0.2 is a Patch Release with a fix for the following area: Sort Query to be a mandatory input for List Job Templates.

-   **Version 1.0.1 - October 2020**

    Patch release of the Ansible spoke for IntegrationHub

-   **Version 1.0.0 - March 2020**

    Automate job scheduling, job templates, inventory and user management in your Ansible Tower environment from the ServiceNow instance.
