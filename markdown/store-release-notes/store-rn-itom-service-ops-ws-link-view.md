# Service Operations Workspace Link View release notes

Version history for the ITOM Service Operations Workspace Link View application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 24.0.2 - May 2024**
    -   Link View is a new feature in the Service Operations Workspace - Express list providing a novel way to visualize tag-based alert groups \(other group types in future versions\). Each alert carries a multitude of attributes, and Link View acts as a cohesive force, illuminating and linking these attributes to shed light on shared elements across different alerts, offering valuable insights into potential root causes.
    -   Link View marks the advent of our first CMDB-less map, transforming how customers approach incident resolution. By visually representing a group of alerts, users can rapidly triage and decode shared attributes in a single glance. This feature represents a leap forward in intuitive alert management.
    -   For users navigating the complexities of IT operations without a mature CMDB or even an initiated discovery process, Link View becomes an indispensable tool. It empowers them to create a visual chart mapping connections within nodes/hosts present in a group of alerts, facilitating efficient troubleshooting and understanding of interdependencies.
    -   The true potency of Link View unfolds when integrated with a mature CMDB. At this juncture, it transforms into a dynamic tool, offering a comprehensive understanding of the impact of alert groups. Link View leverages the CMDB's topology to unveil impacted services, identify probable root causes, establish connections between configuration items \(hosts\), and even activate remediation flows on these items, streamlining incident resolution like never before.
    -   The end goal is to improve the incident response times and troubleshooting efficiency, ultimately transforming the way Operators perform their jobs in the dynamic landscape of IT operations
