# Redox Inbound Integration release notes

Version history for the Redox Inbound Integration application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 6.0.2 - February 2025**

    Starting with the Yokohama release, Redox Inbound Integration is being prepared for future deprecation. It will be hidden and no longer installed on new instances but will continue to be supported. For details on this process, see the Deprecation Process \[KB0867184\] article in the Now Support Knowledge Base.

-   **Version 6.0.0 - August 2024**

    New: Added minor improvements to true up to the Xanadu release.

-   **Version 5.0.0 - February 2024**

    New: Added minor enhancements to true up to the Washington DC release.

-   **Version 4.0.0 - August 2023**

    New: Minor changes to true up for Vancouver release.

-   **Version 2.1.3 - February 2023**

    Minor fixes with data processing.

-   **Version 2.1.2 - December 2022**

    Changed: Upgraded to the updated ServiceNow AI Platform capabilities enabling reduction in the package size of the ServiceNow Store applications with the removal of the redundant folders and their contents.

-   **Version 2.1.0 - November 2022**

    New: Actions were added for providers, scheduling, medications, and patient administration.

-   **Version 2.0.0 - August 2022**

    Hospitals, payers \(insurance\), and life sciences organizations can leverage data from multiple systems and enable bidirectional integrations via the Redox Integration application. Improve inbound and proactive outbound service scalability and capabilities by using the application to integrate with any ONC certified electronic medical records \(EMR\) systems and electronic health records \(EHR\) systems. Additionally, thousands of healthcare systems can leverage their existing Redox integration profile to rapidly adopt the application.
