# Service Reliability Management release notes

Version history for the ITSM Service Reliability Management application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 7.0.0 - June 2026**
    -   Fixed:
        -   Hardened Access Control List \(ACL\) query security.
        -   Improved accessibility with Web Content Accessibility Guidelines \(WCAG\) 2.2 AA.
-   **Version 6.6.2 - May 2026**

    Changed: Version Upgrade to match dependent applications

-   **Version 6.6.1 - March 2026**

    New: Supporting changes for autonomous SLO creation for SRM services

-   **Version 6.5.1 - December 2025**
    -   New: Remove unneeded services from the SRM scope and retire the associated SLOs, keeping your monitoring data relevant.
    -   Fixed: Fixed ability to save notification destination fixes on the Teams page.
-   **Version 6.3.2 - June 2025**

    Fixed: State handling in service checklist guide and Dynamic CI Group linkage for application services.

-   **Version 6.1.4 - June 2025 \(Xanadu\)**

    Fixed: Data mapping box incorrectly showing 'Pending' status after completion.

-   **Version 6.3.1 - May 2025**

    Changed: Standardized integrations through the adoption of AIOps Integration Launchpad

-   **Version 6.2.0 - February 2025**
    -   New:
        -   Added support for Technical Service Offerings in SRM.
        -   Introduced SLO-based change approval policies.
    -   Changed:
        -   Replaced the Alerts tab with Express List in SRM.
        -   Implemented unique naming conventions for edited SLOs.
-   **Version 6.1.3 - February 2025**
    -   Fixed: Access Control List \(ACL\) fixes
    -   New: Updates on the Service overview page
-   **Version 6.1.0 - November 2024**
    -   Fixed:
        -   Incident access via Additional Assignee
        -   Service Checklist Guide addressing broken links and status updates
        -   ACLs and localization issues
-   **Version 6.0.1 - September 2024**

    Fixed: Addressed a configuration issue in scheduled jobs to allow more control over SRM jobs execution.

-   **Version 6.0.0 - August 2024**
    -   New: Support for Tag-based Application services \(min requirement WP6\)
    -   Fixed: Incident Access and Worknotes Visibility \(min requirement WP6\)
-   **Version 4.2.0 - February 2024**

    For ServiceNow customers with teams modernizing into, or already practicing Site Reliability Engineering, the SRM app provides a self serve, guided experience for on-call teams to autonomously manage the health of their technical services using SRE. The experience is built using the Service Operations Workspace application and combines ITOM &amp; ITSM capabilities into a single service operations workflow. Key capabilities include team based access, self serve, on-call scheduling and service level objectives.
