# Compatibility Management release notes

Version history for the Compatibility Management application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 6.6.0 - June 2026**

    Changed: Addressed security directives

-   **Version 6.5.0 - March 2026**

    Fixed: Domain separation changes and read-only field remediation

-   **Version 6.4.0 - December 2025**
    -   Changed: Support added for modules with empty roles
    -   Fixed: Bug around inability to add compatibility rule
-   **Version 6.3.0 - August 2025**

    New: Support for Complex conditions and attributes of Object data type in compatibility rules

-   **Version 6.2.0 - May 2025**

    Fixed: Bug Fix for Deny ACL.

-   **Version 6.1.1 - August 2024**

    Changed: Support for non-alpha numeric characters in Code field for compatibility rule tables.

-   **Version 6.0.1 - May 2024**

    Fixed: Added Code column to compatibility management tables: sn\_compt\_mgmt\_compatibility\_rule, sn\_compt\_mgmt\_compatibles, and sn\_compt\_mgmt\_compatible\_configuration.

-   **Version 5.0.0 - February 2024**
    -   Changed:
        -   Plugin will be installed along with Product Catalog Management Core
        -   Files restructuring to support plugin movement
-   **Version 4.0.0 - November 2023**
    -   New: Ability to carry over the compatibility rules when a copy or new version of a specification is created
    -   Fixed: Removed hardcoded messages to enable i18n
-   **Version 3.0.0 - August 2023**

    No changes.

    Impacted application.

-   **Version 2.0.0 - February 2023**
    -   New:
        -   Support to update related items as part of change and disconnect orders
        -   Support for revision of related items as part of inflight order
-   **Version 1.0.1 - November 2022**

    Changed: Internal optimization to align to NOW engineering standards

-   **Version 1.0.0 - August 2022**

    Create and manage the compatibility rules for defining the horizontal relationships among product, service, and resource specifications. Using compatibility rules orders are fulfilled with required related products, services and resources and store the details of the related product, service and resource inventories after the order fulfilment process.
