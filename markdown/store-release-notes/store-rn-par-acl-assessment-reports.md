# ACL Assessment for Reports release notes

Version history for the ACL Assessment for Reports application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 3.1.2 - February 2025**

    Fixed: 3rd party library update, no functional changes.

-   **Version 3.1.1 - September 2023**

    Fixed: avoid false positives in the scan results.

-   **Version 3.1.0 - May 2023**

    No functional changes.

-   **Version 3.0.1 - March 2023**

    Changed:

    -   Scan through all active RVACL in the instance
    -   The scan is able to identify tables without Report View ACLs and with Read ACLs and to assess if reports and users will be affected
-   **Version 2.0.4 - October 2022**

    Fixed: Set admin as required role when High Security plugin is not active

-   **Version 2.0.3 - September 2022**
    -   Fixed:
        -   False positive results may be generated by assessment scan if report\_view ACL has no roles assigned ​and admin override is set.
        -   Affected users are not identified if report has additional group by columns and user passes only some of the field level ACL's.
        -   Fixes for internationalization.
-   **Version 2.0.2 - August 2022**
    -   Changed: The ACL Assessment for Reports app marks report\_view ACLs as customer customizations when these ACLs are enabled via the app.
    -   Fixed: Most report\_view ACLs no longer marked inactive on upgrade or patch.
-   **Version 2.0.1 - June 2022**

    Fixed: Localization issues on the ACL assessment for Reports info messages

-   **Version 2.0.0 - May 2022**
    -   New:
        -   The security admin can assign users to an existing group and assign the role the report\_view ACL requires from the Affected Users list.
        -   If there are no groups that have at least one of the required report\_view ACL roles, then all groups available in the instance are shown. The security\_admin can select an existing group to assign roles and then users to it.
    -   Changed: Last execution column added to the Affected Reports List.
-   **Version 1.0.6 - November 2021**

    Fixed: Ability to display on the Affected Reports List the database views tables impacted by the Report View ACLs

-   **Version 1.0.4 - October 2021**

    Use the ACL Assessment for Reports to identify reports that users cannot see due to ACLs. The application identifies both the users and the reports. Change the ACL on the table to admit more users, or ignore a user or report to address later.
