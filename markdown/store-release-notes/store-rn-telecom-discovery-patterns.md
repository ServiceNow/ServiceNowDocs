# Telecom Discovery Patterns release notes

Version history for the Telecom Discovery Patterns application on the ServiceNow Store.

**Important:** For details on system requirements and family compatibility, view the application listing on the [ServiceNow Store](https://store.servicenow.com/sn_appstore_store.do#!/store/home) website.

## Version history

-   **Version 2.2.0 - June 2026**

    Telecom Discovery Patterns discover physical network equipment, such as routers and switches, using SNMP and CLI protocols. Discovery runs on a schedule or on demand and captures the basic hardware hierarchy, including cards, slots, subslots, and physical ports. A built-in low-code framework lets you customize discovery based on specific MIBs and automatically updates discovered information in the CMDB.

-   **Version 2.0.0 - March 2026**
    -   Changed:
        -   The life-cycle stage and status now automatically update to Operational and Available if not already set.
        -   The equipment reference is now set as part of discovery.
-   **Version 1.0.2 - December 2025**

    New: A new granular admin role, tsom\_visibility\_admin, has been added to TSOM Core. Users with this role will have the ability to operate the TSOM Patterns application functionality.

-   **Version 1.0.1 - February 2025**

    Telecom Discovery Patterns supports scheduled and On-Demand deep discovery of the Physical Network via SNMP and CLI. It supports direct discovery of Physical Equipment \(Routers, Switches\) and their underlying hierarchy \(Cards, Slots, Sub-Slots, Physical Ports\) using standard SNMP and CLI. It also supports a low code framework to allow custom MIB-based discovery and updating the discovered information into CMDB.
