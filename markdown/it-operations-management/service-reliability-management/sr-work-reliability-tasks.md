# Working with SRM reliability tasks

Alerts, incidents, and change requests are reliability tasks. From creation to resolution, SRM enables you to manage your alerts through the entire response life cycle.

The Reliability Tasks pane contains three tabs:

-   **Alerts** are created within SRM manually or using different integrations. Alerts is the default tab. For more information on alerts, see [Working with alerts in SRM](../concept/sr-work-alerts.md).
-   **Incidents** are created manually or based on SLO and SLI. For more information on incidents, see [Working with Incidents in SRM](../concept/sr-work-incidents.md). For more information on SLO, SLI, and error budget actions, see [Create SLO, SLI, and Error budget policies](../../slo-management/task/sr-create-slo-sli.md).
-   **Change requests** can be created within SRM. See [Manually create an SRM change task](sr-create-change.md).

Responders and above are notified for updates to tasks based on their notification preferences.

-   **[Working with alerts in SRM](../concept/sr-work-alerts.md)**  
Events take the form of alerts under reliability tasks in SRM. From creation to resolution, SRM enables you to manage your alerts through the entire alert life cycle.
-   **[Working with Incidents in SRM](../concept/sr-work-incidents.md)**  
Plan ahead of service disruptions and have SRM send notifications and create status when incidents occur. Distractions are minimized and teams stay focused on remediation.
-   **[Manually create an SRM change task](sr-create-change.md)**  
Create a change task to implement a controlled process for modifying approved and supported actions for SRM.
