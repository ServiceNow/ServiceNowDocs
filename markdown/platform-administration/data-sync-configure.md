# Configuring Multi-instance Management

Learn about the configurations of different instances to communicate data across multiple instances within a customer’s account.

-   [Basic trust configuration for data sync applications](grant-access-other-instances.md): Certain ServiceNow applications have the ability to provide data visibility across instances within a customer’s account.

    **Note:** This ability can be leveraged only if you have Security Center and Subscription Management installed on your instance. See [Security Center](https://www.servicenow.com/docs/access?context=sec-center-v2&version=zurich&pubname=zurich-platform-security&ft:locale=en-US) and [Subscription Management](../../subscription-management/reference/subscription-management-landing-page-v2.md) for more information.

-   [Cross-instance application trust configuration](grant-access-v2.md): Multi-instance management provides a mechanism to streamline the management of trust configurations across your entire multi-instance environment.

-   **[Basic trust configuration for data sync applications](grant-access-other-instances.md)**  
Certain ServiceNow applications have the ability to provide data visibility across instances within a customer’s account.
-   **[Cross-instance application trust configuration](grant-access-v2.md)**  
Multi-instance management provides a mechanism to streamline the management of trust configurations across your entire multi-instance environment.
