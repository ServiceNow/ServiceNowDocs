# Managing Instance Clone

You can use Instance Clone to request, cancel, schedule, or modify clones.

Here are a few basics about clone:

-   [Cancel a clone](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/ai-platform-administration/cancel-clone-cac.md)
-   [Cancel a clone \(legacy\)](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/ai-platform-administration/cancel-clone-cac.md)

    You can cancel a clone that is scheduled, requested, or in progress until the Node Repoint state. Canceling a clone retains the target instance as-is in its pre-clone state with its original data.

-   [Handling a large amount of in-progress update sets](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/ai-platform-administration/clone-update-sets-example.md)

    You can batch a large group of update sets.

-   [Roll back a clone](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown)

    Roll back a clone to remove the latest cloning updates on a cloning target if a mistake was made or an error has occurred.
