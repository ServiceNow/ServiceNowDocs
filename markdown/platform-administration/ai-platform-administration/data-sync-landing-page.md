# Multi-instance Management

Multi-instance Management offers tools and visibility to control how instances share information for enabled applications. This feature can be utilized for ServiceNow applications when they need to communicate data across multiple instances within a customer’s account.

-   [Basic trust configuration for data sync applications](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/ai-platform-administration/grant-access-other-instances.md): Certain ServiceNow applications have the ability to provide data visibility across instances within a customer’s account.

    **Note:** This ability can be leveraged only if you have Security Center and Subscription Management installed on your instance. See  and [Subscription Management](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/subscription-management-landing-page-v2.md) for more information.

-   [Cross-instance application trust configuration](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-administration/ai-platform-administration/grant-access-v2.md): Multi-instance management provides a mechanism to streamline the management of trust configurations across your entire multi-instance environment.

    **Note:** Multi-instance management has new Yokohama platform capabilities that ServiceNow applications can leverage to exchange information among instances. To experience this and for more information, see the Impact Platform Health application.
