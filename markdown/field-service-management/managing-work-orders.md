# Managing work orders and work order tasks

Create work orders, and work order tasks using templates that integrate the right skills and scheduling.

Create, approve, qualify, or close work orders and manage SLAs using the Field Service Management application.

-   **[Managing work orders](../../planning-and-policy/concept/c_ManageWorkOrders.md)**  
Work orders are requests for off-site work. Users with the appropriate roles create the work orders and provide the necessary information needed for the work order tasks. Then other users qualify those work orders and create the tasks necessary to complete the work order.
-   **[Apply a work order template](../../it-services/task/t_UseTheWorkOrderTemplate.md)**  
Work order templates enable users with the proper roles to automatically create work orders, tasks, and part requirements.
-   **[Managing work order tasks](../../planning-and-policy/concept/c_WorkOrderTasks.md)**  
Use work order tasks to define separate activities that must be done to complete a work order.
-   **[Review a task](../task/review-send-task-back.md)**  
As a reviewer, review the details of a work order task. If the details are sufficient, you can close the task. If more information is required, send the task back to the agent so they can add the requested details.
-   **[Execute work order tasks](../../planning-and-policy/concept/c_ExecuteWorkOrderTasks.md)**  
After the dispatcher dispatches work order tasks, the ServiceNow system automatically sends the tasks to the assigned agent's queue.
-   **[Add a knowledge article to a work order or work order task](../task/add-knowledge-workorder.md)**  
Add additional information to complete tasks.
-   **[Record the time worked on a work order task](Recording-time-spent-on-work-order-tasks.md)**  
Manage and record time worked on work order tasks and activities. Time recorded entries automatically generate time cards and time sheets for approval by managers.
-   **[Field Service Management SLAs](../../planning-and-policy/concept/c_FieldServiceAutomationSLAs.md)**  
ServiceNow SLAs track the service level provided by groups and individuals.
