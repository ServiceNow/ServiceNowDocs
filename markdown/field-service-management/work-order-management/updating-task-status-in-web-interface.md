# Updating task status in web interface

Enable dispatchers, agents, and managers to efficiently track and manage the progress of work order tasks, ensuring that everyone involved can easily update and manage the current status.

-   **[Execute work order tasks](../../planning-and-policy/concept/c_ExecuteWorkOrderTasks.md)**  
After the dispatcher dispatches work order tasks, the ServiceNow system automatically sends the tasks to the assigned agent's queue.
-   **[Use drop off lists](../../planning-and-policy/concept/c_UseDropOffLists.md)**  
Agents use drop off lists to return items that are in their personal stockroom.
-   **[Record the time worked on a work order task](Recording-time-spent-on-work-order-tasks.md)**  
Manage and record time worked on work order tasks and activities. Time recorded entries automatically generate time cards and time sheets for approval by managers.
-   **[Create a signed PDF summary for a work order](../task/create-wo-pdf-summary.md)**  
Request that customers digitally sign and confirm a closed work order. A PDF summary of the signed work order is then created.
