# Recall campaign form

The recall campaign form enables the OEM to create a recall campaign.

<table id="table_xbz_nb3_wfc"><thead><tr><th>

Fields

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Recall campaign unique number. This field is automatically generated. The number starts with RCP.

</td></tr><tr><td>

Campaign name

</td><td>

Name of the campaign.

</td></tr><tr><td>

Planned start date

</td><td>

Start date from which the recall campaign is applicable.

</td></tr><tr><td>

Product issue number

</td><td>

Serial number of the product for which the recall campaign is created.

</td></tr><tr><td>

State

</td><td>

Product state. This field is automatically fetched based on the product number.

</td></tr><tr><td>

Recall type

</td><td>

Recall issue type. Options are:-   Product defect
-   Regulatory
-   Safety
-   Technical service bulletin
-   Stop drive recall
-   Park outside advisory
-   Miscellaneous

</td></tr><tr><td>

Priority

</td><td>

Priority of the recall. Options are:-   1-Critical
-   2-High
-   3-Moderate
-   4-Low
-   5-Planning

</td></tr><tr><td>

Assignment group

</td><td>

Group of the recall manager who is responsible for handling the current campaign.

</td></tr><tr><td>

Assigned to

</td><td>

Recall manager from the assignment group who is responsible for handling the current campaign.

</td></tr><tr><td>

Short description

</td><td>

Short description of the product issue.

</td></tr><tr><td>

Recall issue details

</td><td>

Detailed issues for which the product is recalled.

</td></tr><tr><td>

Work notes

</td><td>

Additional notes, if any.
