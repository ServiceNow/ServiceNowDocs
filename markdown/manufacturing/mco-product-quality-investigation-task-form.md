# Product quality investigation task form

Product quality investigation task form enables you to add the investigation case task details.

<table id="table_zf3_l43_xhc"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Product quality investigation case task number is automatically generated. The number starts with PQIT and incremented for every new report.

</td></tr><tr><td>

Parent

</td><td>

Select the parent. It’s used to build the hierarchical categories \(parent&gt;child\).

</td></tr><tr><td>

Priority

</td><td>

Choose the severity of the non-conformance case task from the options:-   Critical
-   Major
-   Moderate
-   Minor

</td></tr><tr><td>

Due date

</td><td>

Select the end date for quality investigation case task.

</td></tr><tr><td>

Short description

</td><td>

Enter a short note.

</td></tr><tr><td>

Description

</td><td>

Enter a detailed description of the product quality investigation.

</td></tr><tr><td>

State

</td><td>

Choose the state of the non-conformance case task.-   Pending
-   Open
-   Work in progress
-   Closed complete
-   Closed skipped
-   Closed incomplete

</td></tr><tr><td>

Assignment group

</td><td>

Select the organizational units or teams responsible for handling specific tasks, cases, or issues

</td></tr><tr><td>

Assigned to

</td><td>

Select the user to whom the non-conformance case task has been assigned.
