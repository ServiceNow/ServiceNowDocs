# Manage documents from Documents tab

The Documents tab lets supplier contacts upload and manage documents without needing a supplier manager to initiate the process first. Supplier contacts can also manage document permissions from the Documents tab.

## Before you begin

Role required: sn\_slm.contact

## Procedure

1.  Navigate to the Supplier Collaboration Portal home page by accessing your instance URL and adding a `/supplier` suffix.

    For example, `https://example.com/supplier`.

2.  In the portal header, select the **Documents** tab.

3.  Upload, download, maintain versions, or manage permissions for documents as follows.

Use this option to upload documents without needing a supplier manager to initiate the process first.

 1.  Select **Create Document**.
2.  Fill in the document details, such as the owner, document classification, and document type.
3.  Upload your document in the Attachments section.
4.  Select **Save**.

**Upload a new version**

</td><td>

Use this option when you want to upload a new version of an existing document:1.  Select a document from the list.
2.  Go to the **Version history** tab.
3.  Select **Add version**.
4.  Upload the new version.
5.  Select **Submit**.


</td></tr><tr><td id="d196987e171">

**Manage document permissions**

</td><td>

Use this option to manage read and write permissions for a document.

 1.  Select the document.
2.  Go to the **Permissions** tab.
3.  Provide read or write access to supplier contacts, as required.

**Note:** The document owner and primary supplier contact are automatically granted read and write access. Secondary contacts are granted read access by default.

</td></tr><tr><td id="d196987e200">

**Download a document**

</td><td>

Download the latest or specific version of a document.

 1.  Select a document.
2.  To download the latest version, select **Download latest attachment**.
3.  To download a previous version, go to the **Version history** tab.
    1.  Select the document version from the version history.
    2.  Select the download icon.
