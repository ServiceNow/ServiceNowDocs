# Manage User Experience

Explore in-depth transactions for a user or a node for a given period.

## Before you begin

**Note:** The functionality is enabled through the Impact Delivery Instance \(IDI\) portal where a customer admin reviews the Terms &amp; Conditions and opts-in via the Manage Permissions page.

Role required: admin

## Procedure

1.  Navigate to **Impact** &gt; **Platform Health** &gt; **Monitor** &gt; **Instance Observer** &gt; **Analytics** &gt; **User Experience**.

2.  Select an instance and a date range from the filter options.

    Select **Update instance default settings** to configure default snapshot settings.

3.  Select **Get Snapshot** to load the User Experience data charts.

    The following data analysis options are available:

    -   Overview
    -   Node
    -   User
    -   URL
    -   Transaction Type
    -   Application Monitoring
    ![Shows the User Experience snapshot.](../image/io-user-ex-snapshot.png)

4.  Drill into each data point to open the particular details.

    For instance, select a user data point on the chart. The User transaction details open with various data points. Select each data point to further drill into each area.

    ![Shows the user transaction details.](../image/io-user-ex-details.png)

5.  Select the **URL** tab to access each specific URL transaction.

    ![Shows the User Experience URL transaction table.](../image/io-user-ex-url-details.png)
