# Using the Access Simulator

Use the Access Simulator to simulate the user's access to the specified table would change once a role or group is assigned or removed.

## Before you begin

Role required: admin, access\_analyzer\_admin

Enable the take actions. For more information, see [Configuring the Access Simulator \(Take actions\)](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-security/identity/configure-access-simulator.md).

## Procedure

1.  Navigate to **All** &gt; **Access Analyzer** &gt; **Access Simulator**.

2.  Select the **Simulate** option under each section based on the following scenarios:

    -   [Adding a Role to the user](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-security/identity/add-role-to-user.md)
    -   [Removing a Role from the user](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-security/identity/remove-role-from-user.md)
    -   [Adding the user to a Group](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-security/identity/add-user-to-group.md)
    -   [Removing the user from a Group](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/platform-security/identity/remove-user-from-group.md)
