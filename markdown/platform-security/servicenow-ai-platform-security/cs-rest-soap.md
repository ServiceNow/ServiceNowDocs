# Sign the REST and SOAP messages in the production instance

Use update sets to sign and validate the REST and SOAP messages by enabling the code signing in production and non-production instances.

## Before you begin

-   Establish Circle of Trust between the production and non-production instances.
-   Role required: security\_admin

-   **[Sign the existing REST and SOAP messages](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/platform-security/servicenow-ai-platform-security/rest-soap-code-sign.md)**  
Sign and validate the existing REST and SOAP messages by enabling the Code Signing in production and trusted non-production instances.
-   **[Sign new REST and SOAP messages](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/platform-security/servicenow-ai-platform-security/cs-new-rest-soap.md)**  
Sign and validate the new REST and SOAP messages from the trusted instance by enabling the Code Signing in production and trusted instances.
