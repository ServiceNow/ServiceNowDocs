# Assign a processing activity to a key stakeholder

Assign a processing activity to a key stakeholder with edit access to let them update it directly from the Employee Center.

## Before you begin

The processing activity privilege of the stakeholder must be set to **Edit processing activity and respond to privacy assessments**.

Role required: sn\_privacy.analyst

## About this task

Assigning a processing activity to a key stakeholder gives a business owner direct access to review and update it. This eliminates the need for relaying changes through the privacy team. The owner updates the activity in the Employee Center and assigns it back to the privacy team when complete.

You can assign a processing activity to a key stakeholder when it is in the **Discover** state.

## Procedure

1.  From the List view of the Privacy Workspace, navigate to **Processing activities** &gt; **Unassigned processing activities**.

2.  Select the processing activity to be assigned to a key stakeholder.

3.  Select the more actions icon \[Omitted image "more-actions-new.png"\], and select **Assign to key stakeholder**.

    **Note:** If a stakeholder is already assigned to the processing activity, select **Reassign to key stakeholder** to change the assignment to a different user. This removes the edit access of the existing stakeholder.

4.  In the **Assign to key stakeholder** window, select a stakeholder.

    **Note:** Only stakeholders with the processing activity privilege, **Edit processing activity and respond to privacy assessments**, are available for selection. To modify privileges, see [Add key stakeholders to a processing activity](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/privacy-workspace/add-stakeholders-to-a-pa.md).

5.  Select **Assign**.


## Result

The assigned stakeholder receives an email notifying them of the processing activity assignment. The stakeholder can navigate to **Employee Center** &gt; **GRC tasks** &gt; **Tasks** &gt; **My pending tasks** &gt; **Processing activities** to view and edit the assigned processing activity record.
