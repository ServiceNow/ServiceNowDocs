# Workflow of Personal Data Rights

A personal data rights request is initiated by a requester and directed to the Personal data rights \(PDR\) agent. The agent then processes the request, assigns it to the appropriate task owners, the task owners work on their tasks, and then request is closed.

The detailed workflow is explained as follows:

1.  A requester requests for information on personal data using one of the following methods.
    -   Email
    -   Phone
    -   Other applications using API integrations
2.  A designated user validates if the user is authentic and genuine and the request is sent to the PDR agent.
3.  The agent then identifies the action task owners to handle the data request.
4.  The action task owners work on the request and notify the agent.
5.  The agent reviews the tasks and then closes the request and notifies the requester.
