# Regulatory change tasks

When a manager or user marks an unassigned alert as applicable, a regulatory change task is created. A regulatory change task is used by the stakeholders to analyze and identify the changes that should be performed on the alerts. The stakeholders can then create action tasks.

A regulatory change task serves as the parent for all the action tasks that have been created to complete the identified changes. For example, an action task can involve making updates to the policies, controls, and risk statements in the regulatory library.

A manager with the sn\_grc\_reg\_change.manager role assigns a regulatory change task to an identified user with the sn\_grc\_reg\_change.user role in the system. The regulatory change task displays the details of the task such as the assignment group, the approver, the assignee, and other relevant information.

After a regulatory change task is completed, the associated regulatory alert can be closed.
