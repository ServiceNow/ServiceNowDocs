# Compliance case task workflow

You can collaborate with multiple teams to investigate, perform an impact assessment, and gather the evidence to capture the details and responses for review in the GRC: Compliance Case Management application.

After a compliance case is reported, the compliance analyst can create case tasks and assign those tasks to the various case task owners. The case task owners can then work on the tasks and submit them for the case analyst to review. There are three types of case tasks:

-   **Assessment**

    An Assessment case task requires the case task owner to take an assessment and provide responses. A case task of this type can only be reassigned.

-   **Investigation**

    An Investigation case task requires the case task owner to provide observations.

-   **Others**

    An Others case task also requires the case task owner to provide observations.


After the case task owners work on a task, they submit the task for the case analyst to review.

The following example displays the life cycle of a case task.

![Life cycle of a case task.](../image/case-task-workflow.png "Case task workflow")

-   **[Create a case task](../task/create-a-case-task.md)**  
Create a case task to investigate, gather evidence, and perform an assessment so that you can analyze and assess a reported case in the GRC: Compliance Case Management application. Each case can have multiple case tasks that you can assign to different owners across multiple teams.
-   **[Work on a case task](../task/work-on-compliance-case-task.md)**  
Accept the case task, provide the details requested by the case analyst, and submit the case task for a review in the GRC: Compliance Case Management application. When a case task is assigned to an assignment group, any user from the assignment group can accept and work on the task.
