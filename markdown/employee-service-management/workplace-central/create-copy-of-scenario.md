# Create a copy of an existing scenario

Create a copy of an existing scenario and make changes. Save time from a creating a scenario from scratch if there are only minor changes to an already created scenario.

## Before you begin

Ensure that the scenario that you want to copy is in one of the following state:

-   Draft
-   Ready

Role required: sn\_wsd\_spcmgmt.space\_planner

## About this task

Clone an existing scenario as it is using the copying process. Instead of making changes from scratch, make changes to an already created scenario by copying it. Copy a scenario that is in either the Draft state or the Ready state.

When you copy a scenario, the copy is created with the same name, but appended with '- Copy 2' to identify it easily.

## Procedure

1.  Navigate to any one of the following locations:

    -   **All** &gt; **Workplace Central** &gt; **Workplace Central**.
    -   **All** &gt; **Scenario Planning** &gt; **My Scenario Plans**
    You can also open Workplace Central from the Employee Center directly. Navigate to **Workspaces** &gt; **Workplace Central**.

    The Workplace Analytics dashboard opens.

2.  On the left pane, select **Space Optimization** icon \(\[Omitted image "space-optimization-icon.png"\] Alt text: Space optimization icon.\).

    The Space optimization dashboard opens.

3.  In the Scenarios list section, at the end of the scenarios list, click **View all**.

    A list of all the scenarios created in the application are displayed.

4.  Select the check box next to the scenario that you want to copy.

5.  On the top-right corner of the page, select the **Copy** option.


## Result

After a few seconds of copying process, the scenario copy is created. To view the copy, refresh the list by selecting the Refresh list icon \(\[Omitted image "refresh-icon.png"\] Alt text:\). The scenario copy is named with the same name as the copied scenario but appended with a '- Copy 2'. For example, Scenario ABC- Copy 2. When you create a copy of an existing scenario, the scenario opens directly in the stack plan view and in the scenario mode of the copied scenario.
