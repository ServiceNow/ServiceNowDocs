# Neighborhood User Assignment Rule form

The Neighborhood User Assignment Rule form specifies conditions based on which users or their profiles must be assigned to the neighborhood.

<table id="table_nxl_2sb_h3c"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Name of the assignment rule.

</td></tr><tr><td>

State

</td><td>

State of the assignment rule record. This field is automatically set to **None** and can't be edited.

</td></tr><tr><td>

Neighborhood

</td><td>

Neighborhood that the rule is being created for.

</td></tr><tr><td>

Active

</td><td>

Option to set the assignment rule to active. This check box is automatically selected.

</td></tr><tr><td>

Building

</td><td>

Building that the assignment rule must be applied to.

</td></tr><tr><td>

Last sync time

</td><td>

Last time the rule was run to sync the neighborhood and the user assignments.

</td></tr><tr><td>

Floor

</td><td>

Floor that the assignment rule must be applied to. You can select a floor only after selecting a building.

</td></tr><tr><td>

Table

</td><td>

Table that is used to select users for the assignments. You can select either of the following options based on your requirement.-   User \[sys\_user\]: Select the system user records for assignments.
-   User Workplace Profile \[sn\_wsd\_core\_workplace\_profile\]: Select Workplace Profiles for assignments.

</td></tr><tr><td>

Filter

</td><td>

Conditions used to filter users from the selected table. For more information about conditions, see [Condition builder](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/c_ConditionBuilder.md).

</td></tr></tbody>
