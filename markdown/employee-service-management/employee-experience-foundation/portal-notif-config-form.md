# Portal notification configuration form

The portal notification configurations form enables you to create a unified notification system for your preferred portal.

<table id="table_vyz_32r_5dc"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Name of the configuration for identification.The name, ideally, should include the name of the portal that is being configured.

</td></tr><tr><td>

Portal

</td><td>

Portal where the unified notification is being configured.

</td></tr><tr><td>

Order

</td><td>

Order of priority for the record.

</td></tr><tr><td>

Active

</td><td>

Option to enable the configuration.

</td></tr><tr><td>

Tray configuration

</td><td>

Enable bell icon: Option to enable the bell icon on the portal.

</td></tr><tr><td>

Tray configuration

</td><td>

Show notifications count: Option to enable view of the number of notifications accumulated, on the bell icon.

</td></tr><tr><td>

Tray configuration

</td><td>

Show unread count: Option to enable view of the unread notifications count.

</td></tr><tr><td>

Tray configuration

</td><td>

Tray actions: **Mark all as read** option to add to the global utility ellipses panel.

</td></tr><tr><td>

Tray configuration

</td><td>

Standard filters: Types of default filters.-   Critical: Marks the notification as critical.
-   Reminder: Sends a reminder notification.

</td></tr><tr><td>

Tray configuration

</td><td>

Notification classification filters: Types of filter classifications available in notifications.-   Comment: Sends a notification when a comment is added.
-   Request: Sends a notification when a request is created or updated.
-   Task: Sends a notification when a task is created or updated.

</td></tr><tr><td>

Card configuration

</td><td>

Card actions: Types of card actions available for notifications.-   Mark as read: Marks all notifications read.
-   Remind me: Sends a reminder notification.
-   Clear notification: Clears all notifications.

</td></tr></tbody>
