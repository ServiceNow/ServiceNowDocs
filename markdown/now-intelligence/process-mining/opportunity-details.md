# Opportunity details page

The Opportunity details page displays details about the improvement opportunities for a project.

From the Improvement Opportunities icon \(![Improvement opportunities icon](../image/icon-imp-opp.png)\) on the side panel, you can choose which improvement opportunity you want to view in detail.

![List of improvement opportunities for the project](../image/opp-det-listofimp.png)

After selecting an improvement opportunity, all details for that improvement opportunity are provided.

![Opportunity details page after generating highlights](../image/opp-det-page.png)

-   The metrics for each improvement opportunity is provided at the top.
-   In the process graph widget, the improvement opportunity pattern in the process is highlighted in red.

    |Improvement opportunity|Description|Highlight|Example|
    |-----------------------|-----------|---------|-------|
    |Ping-pong|Highlights both nodes and arcs in between so it’s clear that the pattern applies to only these nodes and arcs.|Nodes: A, B,and arcs in between|A-B-C-**D-E-D-E**-F|
    |Rework|Highlights only the designated node.|Nodes A|**A**-B-C-**A**-Z-**A**|
    |Extreme duration|A pattern where transitions take significantly longer than the usual duration between the steps. Therefore, both nodes and the transition are highlighted.|Nodes A, B, and arcs in between|A-B-C-**D-E**|
    |Extreme repetition|A pattern where a transition repeats significantly more than the usual repetition range between the steps.|Nodes A, B, and arcs in between|A-B-C-**D-E-D-E-D-E**-G|
    |Extra step|Identifies an extra step in a variant. Therefore, the designated node and incoming outgoing arcs are highlighted.|Node A and incoming arc and outgoing arc|A-B-C-D-E-**Z**-F|
    |Slow transition|Slow transition applies to the handover from one team to another team.|Nodes A, B, and arcs in between|A-B-**C-D**-E-F|
    |Repeating pattern|Includes arcs to highlight the pattern clearly.|Nodes B, C, D, and arcs in between|A-B-C-D-**B-C-D**-E-F-G|

-   Select **Analyze in analyst workbench**, to open Analyst workbench in the context of the improvement opportunity.
-   Select the options in the **Investigate** list to view the details of the records, run a cluster analysis, or to run a root cause analysis.
-   Choose any of the options on **Take action** to:
    -   Set the status of the improvement opportunity.
    -   Create a note, CIM, and an automation request for the improvement opportunity.

## Process inefficiency highlights

You can generate the highlights to view the main reasons for inefficiency. To generate highlights, you must have the Now Assist for Creator role and the Process Inefficiency Highlights skill activated on the instance. For more details, see [Activate Process Inefficiency Highlights skill](../task/activate-highlight-skills.md).

If the Process Inefficiency Highlights skill isn’t activated on the instance, the highlights option isn’t available. However, if the skill is activated, but you don’t have the Process mining for Creator role, then you can only view the highlights that are generated by someone who has the role. You can’t generate any highlights. For more information about how to generate highlights, see [Generate highlights for improvement opportunities](../task/generate-highlights.md).

In the **Highlights** tab, the main reasons for inefficiency related to the selected improvement opportunity are displayed. You can also view the trends, breakdowns, and variants for the selected improvement opportunity.

The key reasons are listed on the highlights section. These reasons are automatically calculated using the ServiceNow’s Now Assist feature.

For more details, see [Process inefficiency highlights](inefficiency-highlights.md).

In addition to the highlights, trends, breakdowns, and variants are also displayed for the selected improvement opportunity.

-   **[Process inefficiency highlights](inefficiency-highlights.md)**  
After identifying the areas that cause the inefficiency, you need to know what exactly the issue is and how to fix it. To do this manually is very time-consuming. The process inefficiency highlights provide the insight into the exact issues for the identified improvement opportunities.
