# Exploring Platform Analytics dashboards

Use Platform Analytics dashboards to access, organize, and share data in a visual format.

## Dashboard overview

You can find all the dashboards that you have permission to access, dashboards that you own, dashboards that you've bookmarked, and dashboards that have been shared with you by navigating to **All** &gt; **Platform Analytics** &gt; **Library** &gt; **Dashboards**.

For more information, see [Dashboards in the library](../concept/find-dashboard-library.md).

## Dashboards for users with the admin roles

The configuration of the Dashboards page and the data access depends on the user roles. Users without admin roles have limited capabilities, as compared to users with the admin roles. Users with the admin roles can perform such actions as deactivating, activating, deleting, or creating categories.

For more information, see [Dashboards for users with admin roles](../concept/dashboards-for-admin-users.md). For more information on the types of admin roles in Platform Analytics, see [Platform Analytics roles](../../par-for-workspace/reference/platform-analytics-roles.md).

## Dashboard components

When you open a dashboard, you can view the components.

![Section of a dashboard. Refer to the following list for the descriptions of the components.](../../par-for-workspace/image/inline-db-components3.png "Inline dashboard components")

1.  Dashboard title

    Displays the dashboard title. Select the arrow to choose a different dashboard.

2.  Refresh information

    Displays the time or duration when the dashboard was last refreshed. Users with admin roles can configure the refresh intervals. For more information, see [Configure Platform Analytics dashboard settings](../task/configure-ac-db-settings.md).

3.  Certification badge

    Whether the dashboard is company-approved and recommended for use.

4.  Category badge

    Terms or phrases that describe one or more dashboard categories. This information is provided by the dashboard creator or the analytics\_categories\_admin.

5.  Refresh button

    Button for refreshing all the visualizations on the dashboard.

6.  Details button

    Button for displaying the ownership and sharing information about the dashboard. You can view the user, groups, roles, and access permissions. For more information, see [Configure Platform Analytics dashboard details](../task/config-db-in-ac.md).

7.  Insights button

    Button for viewing the insights cards. For more information, see [Proactive analytics insights on dashboards](../../par-for-workspace/concept/proactive-analytics.md).

8.  Edit button

    Button for editing the dashboard. Only users with admin roles, edit permissions, and ownership of the dashboard can edit the contents of the dashboard.

9.  More actions button

    Button for creating, duplicating, sharing, printing, bookmarking, clearing the filters for, or deleting the dashboard. For more information on sharing the dashboard information, see [Share a Platform Analytics dashboard](../task/share-db-in-ac.md).

    ![Dashboard sharing dialog that shows who the dashboard has been shared with and whether they can edit or only view it.](../../par-for-workspace/image/db-share-dialog.png "Share Platform Analytics dashboard")

10. Top stage

    Elements that are visible no matter which tab has the focus. Filters in this section apply to visualizations on all of the tabs.

11. Tabs

    Tabs for managing information on your dashboard and keeping related information in the same place.
