# Defining consent policies according to country

Define the tracking consent policies for users within different countries. You can either select the same tracking consent policy for all countries, use the allocated default values, or define different consent policies for individual countries.

## Before you begin

Role required: admin

## About this task

This task details how to set the various tracking consent policy options you have in the User Experience Analytics Consent Policies page. You have an option to return the policies to the default values. The following default tracking consent policies for countries are available.

**Note:** EFTA stands for the European Free Trade Association \(Iceland, Liechtenstein, Norway, and Switzerland\).

-   US users - No Consent required
-   EU, EFTA, and UK users – Explicit Opt-in
-   Non-US and non-EU, EFTA, and UK users – Notice

For a detailed explanation about the tracking policies, see [Types of tracking consent policies in User Experience Analytics](../reference/uxa-tracking-types.md).

## Procedure

1.  Navigate to **All** &gt; **User Experience Analytics** &gt; **Consent Policies**.

    The User Experience Analytics Consent Policies page displays.

    **Note:** Select a policy to view a list of all countries associated with the selected policy.

2.  Perform one of the following actions.
