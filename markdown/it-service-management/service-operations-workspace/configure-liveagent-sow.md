# Live Agent chat in Service Operations Workspace

Service Operations Workspace enables agents to work on any incident created using Live Agent chat.

An application admin can integrate the Live Agent into the ServiceNow application by installing the com.glide.interaction.awa plugin. For more information about installing and enabling a plugin, see [Activate a plugin](https://www.servicenow.com/docs/access?context=t_ActivateAPlugin&version=xanadu&pubname=xanadu-platform-administration&ft:locale=en-US).

After the plugin installation, an inbox icon ![Inbox icon.](../../configurable-workforce-optimization-itsm/image/inbox-icon.jpg) appears in the sidebar of the Service Operations Workspace window.

By default, the status of the agent is Offline. To start the communication, the agent must modify the status to Available.

When the customer reports an incident through Live Agent \(for example, through the Service Portal\), an incident is created automatically in the Inbox. The agent can then work on the incident and chat with the user simultaneously to resolve the issue.

![Incident created by Live Agent chat in SOW.](../image/live-agent-sow.png "Live Agent in SOW")

For more information about Live Agent chat integration, see [Move from Connect Support to Advanced Work Assignment and Agent Chat](https://www.servicenow.com/docs/access?context=migrate-from-connect-support&version=xanadu&pubname=xanadu-servicenow-platform&ft:locale=en-US).
