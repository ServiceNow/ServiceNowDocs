# Create an order header task

Create an order header task manually to track and coordinate activities at the order level that are not tied to a specific domain order or fulfillment task.

## Before you begin

Role required: sn\_ind\_tmt\_orm.order\_agent or sn\_ind\_tmt\_orm.order\_fulfillment\_manager

## About this task

Order agents and fulfillment agents can create order header tasks manually when a task is needed that is not automatically generated by the Customer Order Task Policy. The order number is automatically populated when a task is created from the Order Header Tasks tab and cannot be changed.

Order header tasks remain open until explicitly closed. The order is not marked as completed until all associated order header tasks are closed.

## Procedure

1.  Navigate to **Workspaces** &gt; **CSM/FSM Configurable Workspace**.

2.  Open the order for which you want to create a task.

3.  Select the **Order Tasks** tab.

4.  Select **New**.

    The **Order number** field is automatically set to the current order and is read-only.

5.  On the form, fill in the fields.

    |Field|Description|
    |-----|-----------|
    |Number|This field is automatically set to a unique task number.|
    |Order number|This field is automatically set to the order from which the task was created. Read-only.|
    |Account|Customer account associated with the order.|
    |Primary contact|Primary contact for the order.|
    |State|Current state of the task. Defaults to **Draft**.|
    |Priority|Priority level for the task.|
    |Assignment group|Group responsible for completing the task.|
    |Assigned to|Agent responsible for completing the task.|
    |Short description|Brief description of the task.|
    |Additional comments \(Customer visible\)|Notes visible to the customer.|

6.  Select **Save**.


## Result

The order header task is created and appears in the **Order Tasks** tab.
