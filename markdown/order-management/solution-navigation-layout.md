# Solution navigation and layout

The solution navigation sidebar and layout settings control how buyers move between configurations in a solution session.

Each configuration in a solution displays its own layout, defined by the blueprint associated with that configuration. No additional layout configuration is required to enable the solution navigation sidebar — it appears automatically when child configurations are added during a session.

The header, background style, buttons, and default Product List display come from the solution root layout. This includes currency display settings and any custom branding applied to the root layout.

## Header actions and scope

Header actions in the solution root layout apply across the entire solution:

-   **Save** and **Cancel** apply to the whole solution, not to the currently viewed configuration.
-   **Reset**, **Validate** \(when a validation enrichment exists on the blueprint\), and **Change Layout** apply only to the currently viewed configuration.

## Solution navigation sidebar

The solution navigation sidebar lists all configurable products in the current session, including the root. Buyers can select any item in the sidebar to navigate to that configuration. Buyers can also navigate to a configuration by selecting its configurable product entry in the solution BOM.
