# TM Forum \(TMF\) APIs for SOM

List of ServiceNow® implementation of TM Forum \(TMF\) APIs for Sales and Order Management \(SOM\).

|REST API|Description|
|--------|-----------|
|[Alarm Management Open API](https://www.servicenow.com/docs/access?context=alarm-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Alarm Management Open API provides endpoints to create, update, and retrieve data from the Events \[em\_event\] and Alert \[em\_alert\] tables. The telecommunications Alarm Management Open API \(sn\_ind\_tmf642\) is a ServiceNow® implementation of the TM Forum Open API specification.|
|[Appointment Open API](https://www.servicenow.com/docs/access?context=appointment-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Appointment Open API is a telecommunication API that allows you to interact with the appointment booking application. Use this API to book appointments and search available time slots. It is a ServiceNow® implementation of the Open API TMForum TMF646 Appointment REST API specification and is conformance certified by TM Forum.|
|[Product Catalog Open API](https://www.servicenow.com/docs/access?context=product-catalog-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Product Catalog Open API provides endpoints to create and retrieve product catalogs, product offerings, and product specifications. It is a ServiceNow® implementation of TM Forum TMF620 Product Catalog Management API REST specification.|
|[Product Inventory Open API](https://www.servicenow.com/docs/access?context=product-inventory-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Product Inventory Open API provides endpoints to create and retrieve product inventories. It is aServiceNow® implementation of the TM Forum TMF637 Product Inventory Management API REST specification.|
|[Product Order Open API](https://www.servicenow.com/docs/access?context=tmf622_product_ordering-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Product Order Open API provides endpoints that enable a standardized mechanism for placing product orders. It is a ServiceNow® implementation of TM Forum TMF 622 Product Ordering Management API REST Specification and is conformance certified by TM Forum.|
|[Service Catalog Open API](https://www.servicenow.com/docs/access?context=service-catalog-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Service Catalog Open API provides endpoints to create and retrieve service specifications. It is a ServiceNow® implementation of TM Forum TMF633 Service Catalog API REST specification.|
|[Service Order Open API](https://www.servicenow.com/docs/access?context=service-order-open-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Service Order Open API provides endpoint that enable a standardized mechanism for placing service orders. It is a ServiceNow® implementation of TM Forum TMF641 Service Ordering Management API specification.|
|[Technical Service Qualification Open API](https://www.servicenow.com/docs/access?context=ts-qualification-api&version=australia&pubname=australia-api-reference&ft:locale=en-US)|The Technical Service Qualification Open API provides endpoints to check the availability of resources and services before submitting orders for fulfillment. It is a ServiceNow® implementation of the TMF645 Service Qualification API REST Specification.|
