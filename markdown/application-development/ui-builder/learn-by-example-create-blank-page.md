# Create a blank page

After creating a demo experience, you can create a blank page that you can then build on to define what the experience provides to your users.

This video shows you how to perform the following procedure.

## Before you begin

Role required: ui\_builder\_admin

## Procedure

1.  If you aren't already viewing the experience you created in the previous procedure, navigate to **All** &gt; **Now Experience Framework** &gt; **UI Builder**, and select the experience you created.

    \[Omitted image "experience-main-page.png"\] Alt text: Main page for your new experience

2.  From the main page for your experience, either select **Create new page** or select the plus \(**+**\) sign next to **Pages and variants**.

3.  On the **First, select a template** screen, select **Create from scratch instead**.

    There are templates you can use, but we are going to create a page from scratch.

    \[Omitted image "page-details.png"\] Alt text: Page details including page name, URL path, and page type.

4.  In the **Name** field, type **My test page**, and select **Continue**.

    **Note:** The **URL path** shows the URL that users can navigate to access the page.

5.  Select a URL page type in the **Type** drop-down list.

6.  Select **Looks good** on the next screen.

7.  On the **Tell us about your variant** screen, do not make any changes.

8.  Select **Continue**.

9.  Select the **Build responsive** option \(default\) for greater control of how the page appears at different screen widths.

10. Select **Create**.

    The visual editor opens. Here, you can begin editing your new page.

    \[Omitted image "visual-editor.png"\] Alt text: UI Builder visual editor
