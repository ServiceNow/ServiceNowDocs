# Create an alert rule to generate custom notifications in RPA Hub

Create an alert rule to generate custom notifications for robots, schedules, process jobs, and other RPA Hub related tables.

## Before you begin

You must do this task in the classic environment.

Role required: sn\_rpa\_fdn.rpa\_support\_user, sn\_rpa\_fdn.rpa\_developer, or sn\_rpa\_fdn.rpa\_admin

## About this task

Alert rules are configurations that define conditions under which an alert or a notification must be triggered. These rules help in managing and responding to issues identified in automations. These alert rules specify the condition or set of conditions that must be met for the alert to be triggered. For example, a trigger condition could be a certain error message, a performance metric exceeding a threshold, or the absence of expected data.

## Procedure

1.  Navigate to **All** &gt; **Robotic Process Automation** &gt; **Monitor** &gt; **Alert Rules**.

2.  Select **New**.

3.  On the form, fill in the fields.

4.  Select **Save**.
