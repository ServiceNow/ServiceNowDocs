# Configuring GRC: Metrics

You can configure GRC: Metrics to meet the needs of your organization.

As a metrics administrator, you can configure the following types of metric definitions.

-   [Automated metric definitions](../task/create-metric-definition.md)
-   [Manual metric definitions](../task/create-manual-metric-definition.md)
-   [Calculated metric definitions](../task/create-composite-metric-definition.md)

-   **[Create a manual metric definition](../task/create-manual-metric-definition.md)**  
Create a manual metric definition to collect the data manually for a metric.
-   **[Create an assignment configuration](../task/create-assign-config-mmd.md)**  
Create an assignment configuration for a manual metric definition to have data owners dynamically assigned to metrics.
-   **[Create an automated metric definition](../task/create-metric-definition.md)**  
Create an automated metric definition to collect the data for a metric. A metric definition specifies the method and key properties of the metric, such as the unit, direction, or nature of the metric.
-   **[Create a calculated metric definition](../task/create-composite-metric-definition.md)**  
Create a calculated metric definition by aggregating the automatic metric definitions, the manual metric definitions, and the calculated metric definitions.
-   **[Formula building in a calculated metric definition](formula-building-at-metric-definition-and-entity-level.md)**  
In a calculated metric definition, you can create formulas to perform calculations for any data that you may require.
-   **[Update a metric definition](../task/update-automated-metric-definition.md)**  
Update an existing metric definition to collect the new metrics. You can associate new goals, targets, and entities with the metric definition.
-   **[Create a metric](../task/create-a-metric.md)**  
Create a metric to perform calculations on your data.
-   **[Create a threshold for a metric](../task/create-a-threshold-for-a-metric.md)**  
Create thresholds such as amber threshold and red threshold for metrics and metric definitions. Thresholds refer to predetermined values used to assess the performance of a metric. These thresholds are typically defined based on specific criteria or objectives and serve as reference points to determine whether the measured value or performance meets, exceeds, or falls below the desired level.
-   **[Create a metric unit](../task/create-a-unit.md)**  
Define the units in which the metric scores are displayed. The units can be numbers, percentages, currencies, quantities of time, or any other unit that you define. The commonly used units are provided with the ESG Management application by default.
-   **[Create a unit family](../../esg-management-reporting/task/create-a-unit-family.md)**  
Create a unit family to group similar units together. For example, the various units of distance such as miles and kilometers can be grouped. Unit families help you to organize units during unit conversion.
-   **[Create fiscal calendars](../task/enable-custom-fiscal-year.md)**  
Collect, aggregate, and report data based on your fiscal calendars, which may differ from the standard Gregorian calendar. Global organizations often operate in countries with unique fiscal calendars. By creating fiscal calendars in the ESG Management application, you enable local entities to collect data according to their own schedules.
-   **[Importing historical metric data](importing-metric-data.md)**  
Import historical metric data by using a pre-defined import template with instructions. This process helps with updating and managing metric data within an organization, helping ensure that all data complies with established business rules and maintains integrity throughout the process.
-   **[Convert metric data to a different unit](../task/convert-data-into-any-unit.md)**  
Convert any metric data into a preferred reporting unit.
