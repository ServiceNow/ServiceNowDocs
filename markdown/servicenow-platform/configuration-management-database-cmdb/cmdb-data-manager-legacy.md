# CMDB Data Manager \(Core UI\)

The CMDB Data Manager built on Core UI lets you centrally create, edit, review, publish, and track Data Manager policies and the tasks generated by the policies.

The landing page of the CMDB Data Manager \(Core UI\) provides a dashboard view of policies, excluded CIs, and open policy tasks. Access the open policy tasks that need attention from the management group or from the CMDB Data Manager administrator. Preview those tasks, and then approve or reject to continue the process.

**Important:**

CMDB Data manager on Core UI is now deprecated and no longer supported or available for new activation. CMDB Workspace provides the latest experience for this functionality. For more information, see [Create a CMDB Data Manager policy in CMDB Workspace](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/servicenow-platform/configuration-management-database-cmdb/data-manager-create-policy-wrkspc.md).

Manage policies in the CMDB Data Manager \(Core UI\):

-   [Create a CMDB Data Manager policy \(in Core UI\)](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/servicenow-platform/configuration-management-database-cmdb/create-data-manager-policy.md)
-   [Approve or reject a CMDB Data Manager task \(in Core UI\)](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/servicenow-platform/configuration-management-database-cmdb/review-data-manager-task.md)
-   [Review attestation tasks in CMDB Workspace](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/servicenow-platform/configuration-management-database-cmdb/review-data-manager-attes-task.md)
-   [Manage exclusion lists of CMDB Data Manager \(in Core UI\)](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/yokohama/markdown/yokohama/servicenow-platform/configuration-management-database-cmdb/manage-data-mgr-ci-exclusion-list.md)

To access the CMDB Data Manager \(Core UI\), navigate to **All** &gt; **Configuration** &gt; **CMDB Data Manager**.

**Note:** Dependent CI management processes generate dependent CI policies for which CMDB Data Manager shows policy preview details that aren't relevant.

1.  Select **View Policies** on the Policies tile on the CMDB Data Manager landing page.
2.  Select **All** in the breadcrumbs string.
3.  Select one of the following policies: 'Dependent CI - Deletion', 'Dependent CI - Archive', or 'Dependent CI - Retire'.
4.  Select the Preview Policy tab on the policy form.

The data that appears on the Preview Policy page is irrelevant for any of those policies.
