# Instance Data Replication set up

Request Instance Data Replication \(IDR\) and prepare for IDR by analysing your data and planning your strategy.

-   **[Request an Instance Data Replication subscription](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/request-instance-data-replication.md)**  
 The Instance Data Replication \(IDR\) plugin requires a separate subscription and must be activated by ServiceNow personnel.
-   **[Preparing for Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/prepare-instance-data-replication.md)**  
Before you set up Instance Data Replication \(IDR\), analyze the tables and columns in the producer and consumer instances to replicate data successfully.
-   **[Preserving table hierarchy in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/preserving-table-hierarchy.md)**  
Decide if you want to replicate a parent-child table hierarchy and what strategy to use for replicating the data in Instance Data Replication \(IDR\).
-   **[Upgrading legacy replication sets to V2 in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/upgrading-legacy-replication-sets-v2.md)**  
Improve message processing and replication performance by upgrading your legacy replication sets in IDR to V2.
