# Configuring Instance Data Replication

Configure producer and consumer replication sets, and then modify records or mappings as necessary in Instance Data Replication \(IDR\).

-   **[Create a producer replication set in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/create-p-replication-set.md)**  
Specify the tables and columns on the producer instance that you want to replicate in Instance Data Replication \(IDR\).
-   **[Create a consumer replication set for Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/create-c-replication-set.md)**  
Create a consumer replication set to specify the tables and columns on a consumer instance that receive the producer data in Instance Data Replication \(IDR\).
-   **[Seed a consumer instance for Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/seed-consumer-instance.md)**  
Seed all or some of the tables in your Instance Data Replication \(IDR\) replication sets depending on the needs of your consumer instance.
-   **[Trigger a workflow after replication using Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/post-replication-actions.md)**  
Use platform business rules to trigger workflows after replication using Instance Data Replication \(IDR\).
-   **[Reconfigure a replication set in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/change-replication-set-config.md)**  
If you must change or update your original replication set, reconfigure Instance Data Replication \(IDR\) replication sets on the producer instance.
-   **[Manage consumer access to replication data in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/approve-consumer.md)**  
Approve a consumer's request to access replication data in Instance Data Replication \(IDR\).
-   **[Transform replication data for Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/modify-replicated-data.md)**  
Transform producer instance data for tables or table columns that are named differently on consumer instances in Instance Data Replication \(IDR\).
-   **[Set up bidirectional replication in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/bi-directional-replication.md)**  
Replicate updated data on a consumer instance to the producer instance using bidirectional replication in Instance Data Replication \(IDR\).
-   **[Create a discrete producer replication set for Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/servicenow-platform/instance-data-replication-idr/discrete-producer-replication-set.md)**  
Distinguish consumers for a producer replication set in Instance Data Replication \(IDR\).
