# Using Interaction Management

Most interaction creation takes place in other applications, to control channels. When needed, you can create an interaction from the Interaction application.

For more information about how interaction are created in other applications, see:

-   [Interaction Management in Service Operations Workspace](https://www.servicenow.com/docs/access?context=interaction-sow&version=zurich&pubname=zurich-it-service-management&ft:locale=en-US)
-   [Interaction Management in Source-to-Pay Workspace](https://www.servicenow.com/docs/access?context=interaction-management-in-procurement-operations-workspace&version=zurich&pubname=zurich-source-to-pay-operations&ft:locale=en-US)
-   [Interaction Management in Supplier Lifecycle Operations](https://www.servicenow.com/docs/access?context=interaction-mgmt-supplier-mgr-ws&version=zurich&pubname=zurich-source-to-pay-operations&ft:locale=en-US)
-   [Interaction management in Accounts Payable Operations](https://www.servicenow.com/docs/access?context=interaction-record&version=zurich&pubname=zurich-source-to-pay-operations&ft:locale=en-US)
-   [CSM Interaction record page](https://www.servicenow.com/docs/access?context=csm-interaction-record-page&version=zurich&pubname=zurich-customer-service-management&ft:locale=en-US)

-   **[Create an interaction](../task/create-interactions.md)**  
An interaction is a record of a conversation between a requester and a fulfiller \(real or virtual\). Though most interaction creation takes place in other applications, if needed, you can create an interaction directly from Interaction Management.
-   **[Associate a record with an interaction](../task/associate-record.md)**  
As an agent, you might need to create a record on another table to work on an interaction. You can associate the record with the interaction.
