# Branded landing page for a single instance

A branded landing page is a public web page displayed to users before they log in to their  ServiceNow mobile app. This pre-login page contains a login button in the header or a deep link that navigates users to a specified area of a  ServiceNow mobile app, bypassing the need to select an instance.

Branded landing pages enable you to create a customized page that navigates the user to specific areas on your Now Mobile and Mobile Agent apps. Select whether to have the same or different landing pages for each of the mobile app, based on the specific needs of your users.

**Note:** Branded landing pages are available only when working with a single instance.

\[Omitted image "branded-landing-page.png"\] Alt text: Landing page with a customized sign in button in the mobile app header.

## Use case

A municipality home page contains a button to create a ticket, which navigates the user to complete a record in the Now Mobile app. As this feature only works with a single instance, the system bypasses the need to select an instance and navigates the user directly to the log in page. After logging in, the page to create a ticket displays.

Other examples of actions users can take after logging in include: sending a request, accessing personal information, or viewing a specific screen.

## Customize your mobile app login screen

By default, when a user logs in, the standard mobile app login screen displays. This login screen can be customized by using mobile publishing and mobile theming. For more information, see [Publish mobile apps with custom branding](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/mobile/mobile-platform/mobile-publishing.md) and [Next Experience theming for mobile](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/xanadu/markdown/xanadu/mobile/mobile-platform/explore-ne-theming.md).
