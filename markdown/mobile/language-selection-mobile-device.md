# Selecting languages on your mobile device

There are two areas where you define the language for your ServiceNow mobile apps: the instance language setup and the mobile app language setup.

**Note:** For the optimum user experience, the same language should be used for the instance and the ServiceNow mobile app language.

For a list of supported languages, see [Localization on mobile devices](../concept/localization-mobile-device.md).

## Defining the ServiceNow instance language

You can change the instance language by selecting the relevant language from the drop-down field in the mobile app login screen. If this field isn't displayed, this means that your instance has been preconfigured for a specific language and can't be changed.

![ServiceNow mobile instance login screen with language selection.](../image/mobile-app-login-language.png)

## Defining the mobile app language

You can change the mobile app language in the ServiceNow mobile app itself. The process differs for Android and iOS applications as shown.

## Language page in your ServiceNow mobile apps

<table id="table_ayg_mhr_byb"><tbody><tr><td>

For both the iOS and Android workflows when changing the language for ServiceNow mobile apps you're directed to the Language page in the ServiceNow mobile app. The Language page contains the following fields.

 The **Preferred Language** is the ServiceNow mobile app language and can be configured from this page. To change the language, see Defining the mobile app language.

 The **Instance Language** is display-only and cannot be configured on this page. To change the instance language, see the following section in this topic, Defining the ServiceNow instance language.

**Note:** The **Instance Language** field is visible in the Vancouver release and later.
