# Propose, promote, and link incident records

Propose or promote security incidents as major security incidents when incidents are identified as critical threat to the organization.

Once Major Security Incident Management is installed on your ServiceNow instance, you can handle the transition of security incidents to major incidents as follows:

-   Propose a security incident to major security incident candidate to initiate a review process.
-   Directly promote a security incident to a major security incident without a review process.
-   Reject a security incident proposed as a major security incident.

You can also link a security incident as a child incident to the major security incident so that all related security incidents can be tracked together.

-   **[Promote to a Major Security Incident](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/security-management/security-incident-response/promote-msi.md)**  
Promote a security incident to a major security incident or reject promoted proposals through the Major Security Incident Management \(MSIM\) Workspace.
