# Activate an agentic workflow for Now Assist for Security Incident Response

You must activate the agentic workflows for Now Assist for Security Incident Response from the AI Agent Studio.

## Before you begin

The Now Assist panel must be activated. See [Activate Now Assist panel standard chat](https://www.servicenow.com/docs/access?context=activate-now-assist-panel&version=yokohama&pubname=yokohama-intelligent-experiences&ft:locale=en-US) for more information.

Role required: aia-admin

## Procedure

1.  Navigate to **All** &gt; **AI Agent Studio** &gt; **Overview**.

2.  Select the card for the agentic workflow that you want to configure and activate.

    If you want to modify an agentic workflow, you can [duplicate](https://www.servicenow.com/docs/access?context=clone-aia-usecase&version=yokohama&pubname=yokohama-intelligent-experiences&ft:locale=en-US) it, adjust the settings to suit your specific needs, and activate the duplicate.

    The Guided setup is displayed. The Describe and connect page is read-only.

3.  Select **Continue**.

4.  For Define trigger, select **Continue**.

    The Define trigger page is read-only.

5.  Select the **Display** toggle for the Select Display page so that the conversation with the AI agent is displayed in the Now Assist panel.

6.  Select the arrow icon to expand the section.

    The sn\_si.analyst role is displayed. This role is supported for the agentic workflows in the Now Assist for Security Incident Response application.

7.  To activate the agentic workflow, select **Save and test**.

    You can now invoke the agentic workflows on-demand.

8.  Select **Use case** \(agentic workflow\) and enter the information you want to test and select **Start test**.
