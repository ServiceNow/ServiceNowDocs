# Architectural artifacts version form

Enterprise Architecture \(formerly APM\) enables Enterprise architects to create multiple versions of architectural artifacts and send for approval. There can be only one approved version for each artifact.

## Architectural artifacts version form fields

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Architectural Artifact

</td><td>

Name of the Architectural Artifact.

</td></tr><tr><td>

File type

</td><td>

File type as either a URL or attachment. Do one of the following. -   To upload a file, select  **Attachment**, click the attachment icon \(![Attachment icon](../../../administer/workspace/image/attach-icon.png)\), and then select the file.
-   To link to a document, select  **URL**  and provide the link.

</td></tr><tr><td>

State

</td><td>

State of the artifact version. This field is automatically set to Draft.
